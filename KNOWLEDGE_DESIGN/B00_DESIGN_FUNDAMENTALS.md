# B00: デザインの基盤と原則

## このファイルの目的
デザインの「なぜ」を理解し、単なるパターン適用ではなく**意図を持った設計**ができるようになること。

## 1. デザイナーの役割の変化

### 旧来の役割
「見た目を整える人」＝ 建物の設計図を描く建築家

### 現代の役割
「体験の導き手」＝ ユーザーを目的地へ心地よく導く旅行ガイド

**求められる5つの力:**
1. **UI設計力** - 美しさ × 使いやすさの両立

## 2. 必須の基盤（これを満たさないと「使えない」）

### 2-1. モバイルファースト

**なぜ？**
- インターネット利用の主流はモバイル
- Googleはモバイル版を評価基準（MFI）としている
- 読み込み3秒以上で半数以上が離脱

**具体的な数値:**
| 項目 | 基準値 | 理由 |
|------|--------|------|
| ボタンサイズ | 44px × 44px以上 | 指でタップしやすい |
| フォントサイズ | 16px以上 | 可読性確保 |
| 行間 | 1.5〜1.6倍 | 詰まりすぎない |
| 読み込み時間 | 3秒以内 | 離脱防止 |

### 2-2. アクセシビリティ

**なぜ？**
- 高齢者・障害者を含む「誰でも」使える状態
- SEOにも有利（検索エンジンはテキストを読む）
- 法的要件になりつつある

**具体的な基準:**
- コントラスト比: 4.5:1以上（通常テキスト）
- 画像には代替テキスト（alt属性）
- キーボードだけで操作可能
- フォーカス状態が視覚的に分かる

### 2-3. パフォーマンス

**なぜ？**
- 表示速度はユーザー体験に直結
- Core Web Vitalsは検索順位に影響

**具体的な対策:**
- 画像圧縮（WebP形式）
- 遅延読み込み（Lazy Load）
- 不要なコード削除

## 3. UX/UI設計の原則

### 3-1. 明快さと一貫性

**原則:** ユーザーが考えなくても構造や操作方法がわかる

**実践:**
- サイト全体で配色・フォント・ボタンスタイルを統一
- ナビゲーションは期待される場所に配置
- 同じ機能には同じ見た目

### 3-2. 視覚的階層（ヒエラルキー）

**原則:** 情報の優先順位を視覚で伝える

**手法:**
- サイズ: 重要なものは大きく
- 色: アクセントカラーで強調
- 余白: 重要なものの周りは余白を多く
- 位置: 上・左は優先度が高い

### 3-3. ナビゲーション

**原則:** 「今どこにいるか」「次はどこへ行けばいいか」が瞬時にわかる

**実践:**
- メニュー項目は7±2以内
- パンくずリストの設置
- 現在地のハイライト

### 3-4. フィードバック

**原則:** 操作が成功したか失敗したかがわかる

**実践:**
- ボタン押下時のアニメーション
- フォーム送信後の完了メッセージ
- エラー時の具体的な説明

## 4. 視線誘導パターン

### F型パターン
- テキスト中心のページ
- 左上→右→下に移動しながら左側を重点的に見る
- **用途:** ブログ、ニュースサイト、情報サイト

### Z型パターン
- ビジュアル中心のページ
- 左上→右上→左下→右下とZ字に移動
- **用途:** LP、トップページ、シンプルな構成

### ガットマンダイアグラム
- 左上（最初に見る）→ 右下（最後に見る）
- CTAは右下に配置すると効果的

## 5. グリッドシステムの意味

### なぜグリッドを使うのか？

1. **一貫性と信頼感:** 整列された要素はプロフェッショナルに見える
2. **レスポンシブ対応:** カラム数を変えるだけで異なる画面に対応
3. **効率化:** 配置の判断が減り、制作スピードが上がる

### 「崩し」の意味

完全に規則的なグリッドは単調になる。**意図的にルールを破る（ブロークングリッド）**ことで：
- 特定のコンテンツを強調
- 視覚的なインパクト
- 個性・差別化

**重要:** 崩すためには、まず「守るべきルール」を理解していることが前提

## 6. 成果指標（KPI）との連動

### デザインは「測定可能な機能」

デザインは主観的なアートではなく、**ビジネス成果を達成するための機能的なツール**です。

| 指標 | 意味 | デザインとの関係 |
|------|------|------------------|
| CVR | コンバージョン率 | CTAの効果、導線設計 |
| CTR | クリック率 | ボタンの視認性、コピー |
| スクロール深度 | どこまで読まれたか | コンテンツの配置、興味維持 |
| 滞在時間 | ページへの関心度 | 情報の質、読みやすさ |
| 直帰率 | すぐ離脱した割合 | ファーストビューの効果 |

**重要:** デザインの良し悪しは「かっこよさ」ではなく、これらの数値に基づいて客観的に判断されます。

### CTAの科学

**配置のゴールデンルール:**
1. ファーストビュー
2. メリット提示後
3. お客様の声の後
4. フッター（または追従型）

**マイクロコピーの最適化:**
- ❌「申し込む」
- ✅「無料で相談する」
- ✅「今すぐ5%OFFで試す」

### 2025年トレンド: AIによる動的最適化（DCO）

LPは「静的な1枚絵」から、AIによって相手に合わせて変化する「動的な接客画面」へと進化しています。

**DCO（Dynamic Creative Optimization）とは:**
- ユーザーの属性や流入経路に合わせて、AIがレイアウトやキャッチコピーを自動で最適化
- 例：Google検索からの流入 → ストレートな商品訴求
- 例：Instagram広告からの流入 → 世界観重視のビジュアル

**現時点での実践方法:**
1. 流入元ごとに異なるLPを用意（手動DCO）
2. A/Bテストで勝ちパターンを発見
3. ヒートマップ分析で改善点を特定

## 7. デザイン決定のチェックリスト

### 優先順位順に確認

1. **基盤は盤石か？**
   - [ ] スマホで見やすいか
   - [ ] 読み込みは速いか
   - [ ] 誰でも読めるか（アクセシビリティ）

2. **迷わないか？**
   - [ ] メニューはわかりやすいか
   - [ ] 目的の情報にすぐ辿り着けるか
   - [ ] 今どこにいるかわかるか

3. **目的は明確か？**
   - [ ] 誰に何を伝えるサイトか
   - [ ] どう行動してほしいか
   - [ ] CTAは適切に配置されているか

4. **魅力はあるか？**
   - [ ] ブランドの個性が表現されているか
   - [ ] ターゲットの好みに合っているか
   - [ ] 競合との差別化ができているか

## 8. アナロジー：家づくり

ウェブサイト構築は「家づくり」に似ている。

| 家づくり | ウェブサイト |
|----------|-------------|
| 地盤 | サーバー・表示速度 |
| 入り口 | モバイル対応 |
| 動線 | ナビゲーション |
| 部屋の配置 | 情報構造 |
| インテリア | ビジュアルデザイン |

**どんなに外観がおしゃれでも、地盤が弱く、入り口が狭く、動線が複雑な家は住みたくない。**

まず「快適に住める機能性」を確保した上で、住む人の好みに合わせた装飾を施す。
